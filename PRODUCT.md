# Product

## Register

product

## Users
企业 HR 经理、法务合规专员以及普通劳动者。用户在审核合规性时需要在一个免受视觉噪音干扰、能够高度专注的专业工作流中操作。

## Product Purpose
以极简、安全、专业的方式，对劳动合同进行本地脱敏、元数据提取，并结合法律向量库与多智能体工作流对违规条款进行审查。

## Brand Personality
极简、安全、专业（苹果亮色美学的精致质感、纯净的白银配色、克制的留白与圆角设计、极佳的阅读对比度）。

## Anti-references
- 避免把所有看板图表、历史活动流水和上传审查界面堆叠在单个页面中，造成严重的视觉拥挤与认知负担。
- 拒绝粗糙的暗色背景、低对比度文本（如暗色背景下的黑字）以及非必要的玻璃拟态装饰（AI Slop）。
- 绝不使用侧边粗边框（Side-stripe borders）作为卡片的修饰。
- 拒绝粗糙的表单和堆砌的卡片网格。

## Design Principles
- **白色苹果美学与高读写对比度**：全局以纯白（#ffffff）和银灰（#f5f5f7）为主色调，文字统一采用苹果深黑（#1d1d1f）与次级灰（#86868b），确保无论在何种显示器下都能清晰阅读，消除弱光对比度缺陷。
- **分区专注与信息解耦**：使用简洁的 Tab 或 sidebar 导航将“数据看板大屏”与“合同智能审计工作区”彻底分开，同一时间只做一件事，保持视界清爽。
- **利用 Streamlit 容器原生的 1px 细线边框**：不使用不合规范的 HTML `div` 切分注入（解决 Streamlit 中 HTML 标签自动闭合导致空框的渲染问题），使用原生的 `st.container(border=True)` 并在全局 CSS 中对容器边框、圆角、背景进行苹果亮色化改写。
- **安全与专业权威感**：脱敏逻辑（[USER_ID] 与 [USER_PHONE]）在界面上以冷静的灰色背景和绿色边框呈现，建立用户的专业安全感。

## Accessibility & Inclusion
- 文本与背景对比度需严格符合 WCAG 规范，亮色模式下正文对比度达到 7:1。
- 风险级别标识不能仅依赖颜色（红色/橙色），应同时有文字和徽章形状的配合。
